purpose of the network layer

  • The network layer, which resides at OSI layer 3 provides services that allow end devices to exchange data across a network
  • The network layer uses four processes in order to provide end to end transport
    • Adressing of end devices - ip addresses must be unique for identification purposes
    • Encapsulation - the protocol data units form the transport layer are encapsulatd by adding up header information including source and destination up addresses.
    • Routing - the network layer provides services t direct packets to other networks. routers select the best path for a packet to take to its destination network.
    • De-encapslation - the destination host decencapsulates the packet to see if it matches its own

Encapsulating IP

Network layer PDU = IP packet
transport layer encapsulation - segment header, data
Network layer encapsulation - ip header, data
the ip header stays the same from the source to the destination host
routers implement different network layer protocols concurrently over a network and use the network layer packet header for routing

Characteristics of IP

IP was designed as a protocol with low overhead - it provides only the functions required to deliver a packet from the source to a destination
An ip packet is sent to the destinatio without prior establishment of a connection
IP was not designed to track and manage the flow of packets. These functions if required are performed by other layers - primarly tcp
IP protocols: Frame, ip header, ...

connectionless

Sender doesnt know: if the reciever is present, if the letter arrived, if the reciever can read the letter
the reciever doesnt know: when it is coming
IP is a connectionless protocol:

Best effort Delivery

IP is a best effort delivery protocol:
unreliable becayse it doesnt guarantee that all packets are sent will be recieved.

Media independent

IP packets can travel over different media
ex: copper ethernet, coper serial, optical fiber, wireless
maximum transmission unit(mtu), data link layer tells the network layer the mtu(maximum size of the PDU)

IPV4 packet header

fields containing binary numbers
significant fields: version(4), differentiated services or diffserv(prio), time to live(ttl) limits lifetime of a packet decreased by one of each router along the way, protocol used to identify the next lvl protcol, source ipv4 adress and destination

limitations of ipv4

ip address depletion, internet routing table expansion, lack of end to end connectivity.

Routers

A router is a Computer

router requires a cpu, operating system and memory
cisco routers: branch, wan, service provider
we focus on branch family of routers

Router CPU and OS

cisco routers require a cpu to execute OS instructions including system initialization, routing functions and switching functions

Router Memory

volatile: RAM
non volatile: NVRAM, FLASH, ROM
Volatile memory requires continual power to store information
Non-volatile memory does not require continual power.
RAM: applications, processes and data needed to be executed by CPU
ROM: store crucial operational instructions and limited IOS, ROM is firmware embedded
NVRAM: permament storage for startup config
flash: permament storage for the IOS and other operating system files such as log or backup files

Inside a router

Network and Host Portions

network portion: 192.168.10
host portion: 10
ipv4 address is hierarchical, all devices must have same network Portions

subnet mask

3 ipv4 addresses must be configured on a host
unique ipv4, subnet mask, default gateway

Logcal AND

3 basic operations used to determine the network address: 1 and 1=1, 0 and 1=0, 0 and 0=0, 1 and 0 = 0

prefix length

equals number of bits in the subnet mask set to 1, writen in slash notation, / followed by the number of network bits

Network, host, and broadcast addresses

network address is all 0s, hosts is the rest in between, broadcast adress is all 1s

Dynamic ipv4 address assignment to a host

Dhcp provides ipv4 address, subnet mask, default gateway and other config info, leased for a certain length of time, on powe down its returned for reuse
